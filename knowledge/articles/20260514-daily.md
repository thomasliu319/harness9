## 大模型能力进展

### Needle：将 Gemini 工具调用能力蒸馏至 2600 万参数模型（[Cactus Compute](https://github.com/cactus-compute)）

Cactus Compute 团队发布了 Needle 模型，通过知识蒸馏将 Gemini 的 Function Calling 能力迁移至仅 2600 万参数的超轻量模型中。这一成果的意义不只是参数量的压缩，更在于它证明了"工具调用"这类结构化推理能力可以从通用大模型中解耦并独立蒸馏，从根本上改变了 Agent 工具调用能力的部署方式。

传统观点认为可靠的工具调用需要依赖数十亿乃至数百亿参数的大模型，而 Needle 的出现意味着边缘设备、IoT 设备和移动端都有可能运行具备高质量工具调用能力的 Agent。极低的延迟和完全本地化的推理使得离线 Agent 场景在实际工程中变得可行。该帖子获得 HN 667 热度，是当日社区关注度最高的 AI 技术话题，也将可能激发更多面向特定 Agent 能力的专项蒸馏研究方向。

**关键亮点**

- 将 Gemini 工具调用能力蒸馏至仅 2600 万参数，参数压缩幅度极为显著
- 实现极低延迟、完全本地化的工具调用推理，无需依赖云端大模型
- 知识蒸馏路线证明特定 Agent 能力可高效迁移至专用小模型
- 对边缘计算和端侧 Agent 场景的工具调用能力部署具有里程碑意义

### Delta Channels 技术详解：LangGraph 长运行 Agent 的存储效率突破（[LangChain Blog](https://www.langchain.com/blog/delta-channels-evolving-agent-runtime)）

LangGraph 1.2 引入的 Delta Channels 是 Deep Agents v0.6 存储优化的底层技术实现，本文深入解析了其设计细节。原有全量快照方案在每个 checkpoint 时刻保存完整状态，存储体积随 Agent 执行步数呈 O(N²) 增长，长运行编程 Agent 的状态存储成本极高。Delta Channels 改用增量变更记录，仅存储每步的状态差异，并周期性写入完整快照作为恢复锚点，将存储复杂度降至 O(N)。

在真实编程 Agent 负载下，Delta Channels 实现了 41 倍的 checkpoint 体积压缩，同时保持对现有 Thread 的向后兼容，支持平滑升级。增量存储 + 周期性完整快照的混合策略在效率与可靠性之间取得了良好平衡，其设计思路也可借鉴到其他有状态分布式系统的设计中。

**关键亮点**

- 将 checkpoint 存储复杂度从 O(N²) 降至 O(N)，根本解决长运行 Agent 的存储爆炸问题
- 真实编程 Agent 负载下 checkpoint 体积减少 41 倍，经生产验证
- 增量变更记录 + 周期性完整快照的混合策略，兼顾存储效率与恢复可靠性
- 向后兼容现有 Thread，支持平滑升级

### The Anatomy of an Agent Harness：系统化的 Harness 架构方法论（[LangChain Blog](https://www.langchain.com/blog/the-anatomy-of-an-agent-harness)）

LangChain 博客发表了关于 Agent Harness 概念解剖的深度文章，明确定义了 Harness 的边界：围绕 AI 模型将原始推理能力转化为可工作系统所需的全部代码、配置和执行逻辑。作者通过反向推导方法，从 Agent 目标行为出发，识别出构成 Harness 的四大核心组件——文件系统访问、代码执行沙箱、持久化记忆系统、验证反馈循环——并逐一分析各组件的设计要点和相互依赖关系。

这篇文章的价值在于它将 Harness 工程从隐性实践提升为显性方法论。文章强调 Harness 工程是决定复杂长程 Agent 性能上限的关键，仅依赖更强的基础模型无法弥补 Harness 设计的缺陷。对于正在设计或评估 Agent 基础设施架构的工程师，该文章提供了可直接应用的系统化框架。

**关键亮点**

- 系统定义 Agent Harness 概念：将 AI 模型转化为可工作系统的代码、配置与执行逻辑
- 提炼文件系统、代码执行沙箱、记忆系统、验证循环四大核心 Harness 组件
- 通过反向推导方法论建立 Harness 设计框架，提供可操作的架构指导
- 强调 Harness 工程是优化复杂长程 Agent 性能的关键决定因素
